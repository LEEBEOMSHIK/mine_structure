# 핵심 원칙

이 저장소의 모델링과 Bedrock 패키지 작업에서는 아래 원칙을 우선한다.

1. **처음부터 새로 만들지 말고 현재 모델을 수정**한다.
   - 큐브 수를 무리하게 늘리기보다 텍스처, 색, UV로 퀄리티를 올린다.
2. 목표 퀄리티는 **Bedrock Marketplace에 들어갈 수 있는** 정돈된 완성도다.
   - 구체적인 톤, 색상, 금지 표현은 각 프로젝트 `README.md`와 스펙 문서를 따른다.
3. 프로젝트별 컨셉의 핵심 실루엣이 멀리서도 읽히도록 한다.
   - 가까이서는 픽셀 음영, UV, 텍스처 디테일로 품질을 보강한다.
4. **단일 축 회전만** 사용한다.
   - Bedrock 호환성을 우선한다.
5. 파츠는 영문 네이밍과 의미별 그룹화를 유지한다.
   - 상세 파츠 구조는 각 프로젝트의 스펙 문서를 기준으로 한다.
6. 여러 콘텐츠를 한 add-on에 묶을 때는 콘텐츠 ID, behavior 파일, resource 파일, 문서 경로를 먼저 레지스트리에 등록한다.
   - 프로젝트별 레지스트리는 `projects/<project_id>/content/README.md`를 기준으로 한다.
